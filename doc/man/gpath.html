NAME

gpath − path resolver for OGDL streams

SYNOPSIS

gpath [options] path [filename]

DESCRIPTION

gpath is a structured text file processor that reads an OGDL file or stream and extracts parts of it according to the specified path.

The input file or stream is loaded as a graph in memory, and printed out according to the given path. If ’.’ is given as path, the whole graph is printed to the standard output. Else the path is resolved against the graph and the result printed.

OPTIONS

-d n

Depth of output graph

-n number

Number of spaces used for indentation (default 4).

-r number

Include the root node.

PATHS

The path language used is OGDL Path, as specified in <http://ogdl.org>. Basically, it is a set of tokens separated by dots, where the dots act as edges of the graph.

Examples

Given the file ’network.g’:

eth0 ip 192.168.0.2 mask 255.255.255.0 gw 192.168.0.1

dns 192.168.0.10, 192.168.0.11

$ gpath eth0.ip network.g

192.168.0.2.

$ gpath -d 1 eth0 network.g

ip mask gw

$ gpath dns.[0] network.g

192.168.0.10

BUGS

Only trees are currently supported; cycles are not yet stabilized in the OGDL specification.

AUTHOR

Rolf Veen. See <http://ogdl.org> for more information.
